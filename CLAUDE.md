# CLAUDE.md

This file provides guidance to Claude Code when working with code in this repository.

## Project

Personal website for jensfischer.nyc, built with Fumadocs (fumadocs-mdx + fumadocs-core) on Next.js App Router with Chakra UI v3.

## Commands

- `npm run dev` — Start dev server
- `npm run build` — Production build
- `npm start` — Serve production build

## Structure

```
src/
  app/
    layout.tsx              — Root layout (html/body, Provider, SiteHeader)
    (home)/
      page.tsx              — Homepage (plain React, not MDX)
      about/
        page.tsx            — About page with clients grid
    knowledge/
      layout.tsx            — Knowledge layout with sidebar
      [[...slug]]/
        page.tsx            — Catch-all route for MDX content
  components/
    site-header.tsx         — Modular header with logo, nav, mobile nav, actions
    docs-sidebar.tsx        — Sidebar navigation with collapsible folders
    docs-content.tsx        — Content wrapper with TOC integration
    docs-toc.tsx            — Table of contents sidebar
    theme-toggle.tsx        — Dark/light mode toggle
    ui/
      provider.tsx          — ChakraProvider + next-themes ThemeProvider
      styled.tsx            — Styled host elements (chakra() is client-only)
      page-container.tsx    — Page width container
      button.tsx            — Button recipe
  theme/
    index.ts                — Chakra system: tokens, semantic tokens, globalCss
  lib/
    source.ts               — Fumadocs page tree loader
  mdx-components.tsx        — MDX component overrides for styling
content/
  knowledge/
    meta.json               — Root knowledge navigation config
    *.mdx                   — Content pages with frontmatter
    tech/                   — Tech & AI notes
    product/                — Product notes
    misc/                   — Misc notes (chess, finance, fitness, …)
  showcase/                 — Showcase project entries (served at /showcase)
source.config.ts            — Fumadocs MDX configuration
next.config.ts              — Next.js config with fumadocs-mdx plugin
```

## Key Patterns

- Uses Fumadocs for MDX processing: `fumadocs-mdx` handles content, `fumadocs-core` provides page tree, TOC, and navigation utilities.
- Content lives in `content/knowledge/` as `.mdx` files with YAML frontmatter (`title`, `description`), served at `/knowledge`. Navigation order is configured via `meta.json` files.
- The `source.config.ts` at the root defines the knowledge collection. The `.source/` directory is auto-generated by `fumadocs-mdx` (run via `postinstall`).
- The `@/.source/*` path alias in tsconfig maps to the auto-generated `.source/` directory.
- Homepage and About are standalone React pages, not MDX.
- Styling is Chakra UI v3 style props. The design system lives in `src/theme/index.ts`: the oklch palette is exposed both as Chakra's own semantic tokens (`bg`, `fg`, `border`) and under its original names (`background`, `foreground`, `muted`, `primary`, `accent`, `ring`, …), and breakpoints/radii/text styles are aligned to the scale the markup was authored against.
- `chakra()` and `chakra.*` are client-only. Server components must import pre-styled elements from `@/components/ui/styled` rather than calling the factory.
- Dark mode via `next-themes` (`attribute="class"`), which Chakra's `_dark` condition reads.
- Site header is modular: `SiteHeader`, `SiteHeaderLogo`, `SiteHeaderNav`, `SiteHeaderNavLink`, `SiteHeaderActions`, `SiteHeaderMobileNav`, `SiteHeaderMobileNavLink`.
