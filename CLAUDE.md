# CLAUDE.md

This file provides guidance to Claude Code when working with code in this repository.

## Project

Personal website for jensfischer.nyc, built with Fumadocs (fumadocs-mdx + fumadocs-core) on Next.js App Router with Tailwind CSS.

## Commands

- `npm run dev` — Start dev server
- `npm run build` — Production build
- `npm start` — Serve production build

## Structure

```
src/
  app/
    layout.tsx              — Root layout (html/body, ThemeProvider, SiteHeader)
    globals.css             — Tailwind CSS v4 styles + CSS custom properties
    (home)/
      page.tsx              — Homepage (plain React, not MDX)
      about/
        page.tsx            — About page with clients grid
    docs/
      layout.tsx            — Docs layout with sidebar
      [[...slug]]/
        page.tsx            — Catch-all route for MDX content
  components/
    site-header.tsx         — Modular header with logo, nav, mobile nav, actions
    docs-sidebar.tsx        — Sidebar navigation with collapsible folders
    docs-content.tsx        — Content wrapper with TOC integration
    docs-toc.tsx            — Table of contents sidebar
    theme-toggle.tsx        — Dark/light mode toggle
  lib/
    source.ts               — Fumadocs page tree loader
    utils.ts                — cn() utility (clsx + tailwind-merge)
  mdx-components.tsx        — MDX component overrides for styling
content/
  docs/
    meta.json               — Root docs navigation config
    *.mdx                   — Content pages with frontmatter
    claude/                 — Claude AI knowledge section
    chess/                  — Chess openings section
source.config.ts            — Fumadocs MDX configuration
next.config.ts              — Next.js config with fumadocs-mdx plugin
```

## Key Patterns

- Uses Fumadocs for MDX processing: `fumadocs-mdx` handles content, `fumadocs-core` provides page tree, TOC, and navigation utilities.
- Content lives in `content/docs/` as `.mdx` files with YAML frontmatter (`title`, `description`). Navigation order is configured via `meta.json` files.
- The `source.config.ts` at the root defines the docs collection. The `.source/` directory is auto-generated by `fumadocs-mdx` (run via `postinstall`).
- The `@/.source/*` path alias in tsconfig maps to the auto-generated `.source/` directory.
- Homepage and About are standalone React pages, not MDX.
- Dark mode via `next-themes` with CSS custom properties (oklch color space).
- Site header is modular: `SiteHeader`, `SiteHeaderLogo`, `SiteHeaderNav`, `SiteHeaderNavLink`, `SiteHeaderActions`, `SiteHeaderMobileNav`, `SiteHeaderMobileNavLink`.
